# Slide Title: Introduction & Problem Statement
# Key Points:
- Coding is challenging for beginners, especially understanding cryptic compiler errors.
- Traditional IDEs throw generic syntax faults that don't explain *how* to fix the issue.
- Existing tools often overlook critical security vulnerabilities (like SQL Injection or Buffer Overflows) unless complex plugins are configured.
- Evaluators and instructors need a tool to quickly analyze student code for logical traps.
# Speaker Notes: Welcome everyone. Today we are presenting the "AI Lexical & Syntax Assistant." The main problem we observed is that beginners struggle with standard compiler errors because they don't provide actionable feedback. Furthermore, basic setups lack built-in security screening, allowing dangerous code execution.

---

# Slide Title: Limitations of Standard AI Code Checkers
# Key Points:
- Pure Deep Learning models (like LSTMs or Transformers) are great at recognizing structural patterns.
- However, AI models alone suffer from hallucinations or unpredictable logic.
- They struggle with "Semantic Errors" (e.g., syntactically correct code that assigns a float to an integer).
- They often fail to catch strict, rule-based security vulnerabilities accurately.
# Speaker Notes: While AI is powerful, a pure AI approach has limits. A neural network might see perfectly structured code and pass it, failing to realize a floating-point number is being crammed into an integer variable, or that a malicious system command is hiding in the code.

---

# Slide Title: Our Solution: The Hybrid Architecture
# Key Points:
- We developed a "Hybrid" detection engine combining Machine Learning with rigorous Static Analysis.
- Layer 1: Security Scanner (Blocks malicious threats immediately).
- Layer 2: Deep Learning LSTM Engine (Analyzes the code sequence for structural syntax bugs).
- Layer 3: Semantic Fallback Engine (AST Parser & exact Regex rules to catch logical/semantic traps).
# Speaker Notes: Our solution is a Hybrid Architecture. Before we trust the AI, we pass code through a rigorous security scanner. Then, an LSTM neural network analyzes the structure. Finally, a static fallback engine serves as the absolute ground truth to catch semantic traps the AI misses.

---

# Slide Title: System Architecture Pipeline
# Key Points:
- User inputs C or Python code via the web interface.
- 1. Code routes to the Security Scanner. If a threat is found, compilation is blocked.
- 2. Safe code routes to the Neural Network for syntax token prediction.
- 3. Safe code also routes to the Semantic Fallback Engine for rule checking.
- 4. The Correction Module synthesizes the data and generates an English AI suggestion and an auto-corrected code snippet.
# Speaker Notes: This is our data flow. Code input -> Security Check -> AI Prediction -> Static Verification -> Final Output. This ensures 100% safety and accuracy before the user sees the final report.

---

# Slide Title: The Deep Learning Engine (LSTM)
# Key Points:
- Built using PyTorch.
- Trained on hundreds of artificial C and Python buggy code snippets.
- The model tokenizes the input code into sequences and padding to a maximum length.
- It outputs exactly whether the code is "Buggy" and predicts the exact token index of the error.
# Speaker Notes: Our AI engine uses a Long Short-Term Memory (LSTM) network built in PyTorch. It tokenizes source code, pads it, and predicts both binary classification (is it buggy?) and sequence localization (where is the bug?).

---

# Slide Title: The Semantic Static Engine (Ground Truth)
# Key Points:
- AI is not enough. We built a custom parser to enforce strict logic.
- Automatically leverages Python's Abstract Syntax Tree (`ast`) for flawless Python checking.
- Uses over 15 rigorous regex patterns for C to catch edge cases.
- Catches: Mismatched brackets, misspelled main functions, unclosed strings, and float-to-int assignments.
# Speaker Notes: Because AI isn't perfect, our Semantic Static Engine acts as the absolute ground truth. For Python, it uses the AST compiler. For C, it uses over 15 deep regex rules to catch complex logic traps like assigning a float to an int, which is grammatically correct but logically flawed.

---

# Slide Title: 100% Secure: The Vulnerability Scanner
# Key Points:
- Scans code for 12 distinct security vulnerability classes before execution.
- Catches: OS Command Injections (`os.system`), Destructive File Ops (`rm -rf`), Buffer Overflows (`gets()`), and Format String Attacks (`printf(str)`).
- Blocks Execution: Prevents dangerous code from ever reaching the compiler.
- Provides actionable AI suggestions (e.g., "Use Parameterized Queries instead of String formatting to stop SQL Injection").
# Speaker Notes: Security is paramount. Our app actively scans code against 12 vulnerability classes—blocking everything from OS injections to Buffer Overflows in C. It doesn't just block it; it tells the student exactly *why* it's dangerous and how to fix it securely.

---

# Slide Title: Intelligent Code Correction
# Key Points:
- The Correction Module translates complex backend errors into plain English.
- Identifies the exact buggy line.
- Provides a clear "AI Suggestion" (e.g., "Missing semicolon ';' at the end of the statement").
- Provides a live "Auto-Corrected Version" of the broken line for the student to copy.
# Speaker Notes: Finding the error is only half the battle. Our correction module takes the buggy line, analyzes the error type, and outputs a plain-English suggestion alongside a fully auto-corrected line of code, making it the perfect learning tool.
